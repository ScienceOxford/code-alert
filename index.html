Setting up the radio

The ability to transmit information using radio waves is built into the micro:bit. The means that micro:bits can send to, and receive information from, other micro:bits!

For this activity you have two micro:bits, and you will put the same program on both of them.

Recreate the image below using the Radio and Basic menus. Scroll down for instructions and debugging tips, or skip to the next step once your new code is working correctly!


on start

So that your micro:bits only listen out for messages from each other (not everyone else in the room!), you will set a group number. If you are in our workshop, use your laptop's number.

Find the radio set group block in the Radio menu, and put it inside your on start loop. Remember to change the group number from 1 to your group!

Debugging tip! If you get weird messages, you might be receiving someone else's code! Double-check your group number.


forever

Next, set your micro:bits to send out a message on repeat.

Find the radio send string block in the Radio menu, and put it inside your forever loop. Change the message to whatever you like!

It is a good idea to slow the messages down a bit - otherwise your micro:bit will be sending messages so quickly that the other micro:bit might get overwhelmed and give you an error message.

Find the pause block in the Basic menu and add it to your forever loop.

The pause is in milliseconds - 100ms is 1 tenth of a second, so your micro:bit will still send messages very quickly.

Debugging tip! If you have other code running as well as the radio code, lots of things might be happening on the micro:bit's display too quickly for you to see properly. Either remove this other code or make your radio pause longer.


on radio received

Now you can tell the micro:bit to do something with the messages it receives.

Find the on radio received (receivedString) loop from the Radio menu. There are three very similar blocks so look carefully to chose the right one.

Add a show string block from the Basic menu.

Instead of displaying a message you type into the show string block, we want it to display the radio message, which is receivedString.

Click on red receivedString oval from the on radio received loop, and drag into position - this will copy the block.

Debugging tip! If your message is not showing in the emulator, check that you are using receivedString, not receivedNumber, and that in the previous step you are using radio send string not send number. If they do not match up, the micro:bit will not interpret the messages the way you expect.


Check your new code on your micro:bits before moving on - do they both show the string that you asked them to send?
